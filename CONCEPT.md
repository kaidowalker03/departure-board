# 発車標シミュレーター 構想メモ

## コンセプト

駅の発車案内表示器（発車標）をWeb上でリアルタイムに再現するサイト。
メインターゲットは京急のパタパタ（ソラリー式）発車標の再現。

## 目標

- 京急に昔あったパタパタ発車標をWebで忠実に再現する
- 静的ダイヤに基づき、現在時刻に連動して次発・次々発を表示する
- 将来的に遅延情報の反映も視野に入れる
- AdSenseによる収益化を目指す

## 対象範囲

- 駅・路線は指定ベースで追加していく
- 表示スタイルは対象によって変える（パタパタ、LED、LCD等）
- まずは京急パタパタを最優先で完成させる

## 京急パタパタ表示器の特徴

- 黒背景に白文字（行先）
- 種別は色付き背景
  - 赤: 快特
  - 青: 特急
  - 黄: 急行（エアポート急行）
  - 緑: 普通
- 表示項目: 時刻 / 種別 / 行先 / 備考（乗車位置等）
- 2〜3行表示（次発・次々発）
- フリップ時に上→下へパタパタ回転するアニメーション

## 方針決定事項

- 街道を征くサイト（WordPress）とは完全に独立した別サイトとして構築する
- Next.js + Vercel で新規構築（A案で確定）

## 技術スタック

| レイヤー | 選定 | 理由 |
|---|---|---|
| フレームワーク | Next.js (App Router) | SSG対応、SEO有利、AdSenseと相性良い |
| 言語 | TypeScript | 型安全 |
| スタイリング | Tailwind CSS + CSS Modules | フリップアニメーションはCSS3 transformで実装 |
| データ | JSON静的ファイル | 駅別時刻表データ。ビルド時にページ生成 |
| ホスティング | Vercel | 無料Hobbyプラン。Next.jsとの親和性◎ |
| バージョン管理 | GitHub | Vercel連携でpush時自動デプロイ |
| 遅延情報(将来) | ODPT API / 京急公式 | API Routeで中継 |

## フリップアニメーション実装方針

- 各文字パネルを上半分・下半分に分割
- CSS `rotateX` で上半分が手前に倒れる → 下半分が現れるアニメーション
- 「現在表示」と「次の表示」の2枚重ね構造
- パタパタ音のSE再生もオプションで検討

## SEO・収益化戦略

- 各駅のページを個別に生成（SSG）
  - 例: `/keikyu/shinagawa`, `/keikyu/yokohama`
  - 「京急 品川 発車標」「京急 横浜 時刻表」等で検索流入
- 駅×路線の掛け合わせでページ数を自然に増やせる
- AdSense申請に必要なもの:
  - 独自ドメイン（年1,000〜2,000円）
  - プライバシーポリシーページ
  - お問い合わせページ
  - ある程度のコンテンツ量（10ページ以上目安）

## メモ

- 京急はODPT（公共交通オープンデータセンター）に参加していないため、時刻表データは手動作成が基本
- Wikipedia等で過去のダイヤ情報を参照可能
- 将来的にGTFSデータが公開されれば自動化も可能
